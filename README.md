# محبار (Mihbar) — منصة نقاشات وتواصل اجتماعي تفاعلية مصقولة

> **ملاحظة ترحيل (Next.js Migration):** هذا المشروع انتقل من React + Vite إلى
> **Next.js 15 (App Router)**. البنية الفعلية الحالية موثّقة في التعليقات
> داخل `src/app/`، `src/MihbarShell.jsx`، و`src/utils/dataLayer.js`. القسم
> أدناه ("البنية المعمارية وهيكلية المجلدات") يوثّق نسخة Vite السابقة تاريخيًا
> ولم يُحدَّث بعد بالكامل ليعكس بنية `app/` الجديدة — راجع الأقسام اللاحقة منه
> بحذر فيما يخص نقاط الدخول (`index.html`/`main.jsx`/`App.jsx` لم تعد موجودة).

منصة **محبار (Mihbar)** هي تطبيق ويب تفاعلي متكامل ومبني باستخدام **React** و**Vite**، يعيد تقديم تجارب نقاشات المنتديات والتواصل الاجتماعي بأسلوب حديث وسلس، مع تصميم بصري مذهل ومريح للعين يعتمد على **Tailwind CSS**.

---

## 🎨 الرؤية البصرية والتصميم
يتميز **Inkore** بتصميمه العصري ذي التباين العالي الذي يدمج بين الخطوط التقنية والألوان الراقية:
- **تنسيق مرن وأنيق**: استخدام خطوط متناسقة تمنح تجربة قراءة فائقة الراحة.
- **تفاعل ديناميكي**: تأثيرات حركية خفيفة وتغذية راجعة (Feedback) لحظية عند التفاعل مع المنشورات أو التعليقات أو التنقل بين الصفحات.
- **واجهة مستخدم متجاوبة (Responsive Design)**: تجربة خالية من العيوب تم تصميمها لتناسب الهواتف الذكية والأجهزة اللوحية والشاشات المكتبية العريضة بسلاسة تامة.

## 🏗️ البنية المعمارية وهيكلية المجلدات
تم تقسيم الشيفرة البرمجية بشكل احترافي ووحدوي (Modular Architecture) لتسهيل التطوير، والصيانة، والتوسيع في المستقبل. إليك تفاصيل الهيكل:

```
├── index.html                  # نقطة الدخول الرئيسية لمتصفح الويب (تتضمن رأس الصفحة وإعدادات العرض)
├── vite.config.js              # ملف إعدادات أداة البناء السريعة Vite
├── postcss.config.mjs          # إعداد معالجة CSS الإضافية ودعم Tailwind CSS
├── package.json                # يحدد الحزم والاعتماديات ومخطوطات التشغيل (Scripts)
├── vercel.json                 # إعدادات النشر المباشر والسلس على منصة Vercel
├── src
│   ├── main.jsx                # نقطة الانطلاق الفعالة لـ React، تضمين التنسيقات العامة والـ StrictMode
│   ├── App.jsx                  # المتحكم الرئيسي بالتطبيق، إدارة الحالة الشاملة والموجه الداخلي (Routing)
│   ├── index.css               # ملف التنسيق العام والمدخل لاستيراد Tailwind CSS
│   │
│   ├── components/             # المكونات الفرعية القابلة لإعادة الاستخدام (UI Components)
│   │   ├── ActionMenuButton.jsx # زر قائمة الخيارات الإضافية للمنشورات (تعديل، حذف، مشاركة، بلاغ)
│   │   ├── BackButton.jsx       # زر العودة الذكي ذو التصميم المتناسق للتنقل بين الصفحات
│   │   ├── ConfirmDialogs.jsx   # النوافذ المنبثقة لتأكيد الإجراءات الحساسة (مثل حذف منشور أو إلغاء تعديل)
│   │   ├── Icons.jsx            # مكتبة الأيقونات المركزية للتطبيق لضمان الاتساق البصري
│   │   ├── MarkdownAttachments.jsx # معالج وعارض المرفقات الغنية داخل المنشورات
│   │   ├── MdEditorPage.jsx     # صفحة المحرر المتقدمة التي تدعم لغة التنسيق Markdown مع معاينة حية
│   │   ├── FeedItem.jsx         # بطاقة عرض المنشور التفاعلية في القوائم والصفحة الرئيسية
│   │   ├── PostInput.jsx        # حقل إدخال وكتابة منشور جديد مع خيارات التنسيق السريع والمرفقات
│   │   ├── ProfilePage.jsx      # صفحة الملف الشخصي للمستخدم، تحتوي على الإحصاءات والأوسمة والنشاط الشخصي
│   │   ├── ReactionRow.jsx      # شريط التفاعلات التعبيري (أعجبني، ضحك، حب، إلخ) مع عدادات حية
│   │   ├── SettingsPage.jsx     # لوحة التحكم بالإعدادات (تخصيص المظهر، إدارة البيانات، إحصاءات الجهاز)
│   │   └── ThreadView.jsx       # الواجهة التفصيلية المخصصة لعرض المنشور الفردي مع شجرة التعليقات المتداخلة
│   │
│   ├── constants/
│   │   └── index.js            # الثوابت العامة للمشروع (مثل تصنيفات المنشورات، الإعدادات الافتراضية، والحدود القصوى)
│   │
│   ├── data/
│   │   └── seeds.js            # البيانات التجريبية الأولية (Seed Data) لملء الموقع بتفاعلات حية عند التشغيل لأول مرة
│   │
│   ├── hooks/
│   │   └── useBreakpoint.js    # خطاف مخصص (Custom Hook) لتتبع مقاسات الشاشات وتعديل الواجهة برمجياً
│   │
│   ├── locales/
│   │   └── strings.js          # ملف الترجمة والنصوص لتسهيل دعم اللغات المتعددة (يدعم العربية بشكل أساسي)
│   │
│   └── utils/
│       └── index.js            # وظائف مساعدة عامة (تنسيق التواريخ، توليد الهاش، وتحليل نصوص Markdown)
```

## 🛠️ تحليل تفصيلي لأبرز الملفات والمكونات

### 1. `src/main.jsx` & `src/index.css`
- **`main.jsx`**: يستورد المكتبات الأساسية لـ React ويقوم بتركيب المكون الأب `App.jsx` داخل عنصر الـ DOM الذي يحمل المعرف `#root`. كما يربط ملف التنسيق العام `index.css`.
- **`index.css`**: يعتمد على محرك `@import "tailwindcss";` لبناء وتوليد الفئات الديناميكية المساعدة، مما يضمن خفة حجم التنسيقات وسرعة تحميلها.

### 2. المتحكم الرئيسي `src/App.jsx`
يقوم بدور العمود الفقري للتطبيق حيث يدير:
- **إدارة الحالات (State Management)**: تخزين قائمة المنشورات، التعليقات، بيانات المستخدم الشخصية، والتفضيلات العامة وتحديثها بشكل فوري.
- **نظام التوجيه الداخلي (Client-Side Router)**: محاكاة تنقل الصفحات بسلاسة متناهية (الرئيسية، تفاصيل المنشور، الملف الشخصي، المحرر، الإعدادات) دون الحاجة لإعادة تحميل الصفحة.
- **نظام الأمان والتحكم اللامركزي**: يتضمن آليات حظر الأجهزة غير المرغوبة وتقييد معدل النشر لحماية بيئة النقاش.

### 3. مجلد المكونات `src/components/`
- **`ThreadView.jsx`**: يمثل واجهة نقاش عميقة، حيث تظهر الردود والتعليقات بشكل متداخل شجري، مع إمكانية الرد على أي تعليق فرعي ومشاركة الروابط المباشرة.
- **`FeedItem.jsx` & `PostInput.jsx`**: يعرضان المحتوى بطريقة كاردات عصرية تتضمن معلومات الكاتب، تاريخ النشر، شارات التصنيف، والمرفقات. ويسهل `PostInput` عملية الكتابة بلمسات جمالية مريحة.
- **`MdEditorPage.jsx`**: يوفر محرر نصوص ثنائي الأقسام (كتابة ومعاينة في نفس الوقت) يتيح للمستخدم صياغة نصوصه المتقدمة باستخدام لغة الـ Markdown ورفع المرفقات بسهولة.
- **`ProfilePage.jsx`**: مساحة مخصصة لعرض إنجازات المستخدم ونشاطه التاريخي، مجهزة برسوم بيانية مبسطة ومؤشرات تفاعل وأوسمة يحصل عليها تلقائياً بناءً على نشاطه.

### 4. المساعدات والثوابت `src/utils/` & `src/constants/`
- **`utils/index.js`**: يحتوي على محرك مخصص لتحويل الـ Markdown إلى HTML آمن ونظيف، ووظائف لتوليد بصمات الأجهزة الفرعية وتنسيق الفوارق الزمنية الذكية (مثال: "قبل ٥ دقائق").
- **`constants/index.js`**: ينظم خيارات النظام وقواعد تصفية المحتوى والكلمات المحظورة، مما يسهل تعديل سلوك التطبيق بالكامل من مكان واحد.

## 💾 معالجة وتخزين البيانات (LocalStorage Layer)

يعتمد التطبيق في إصداره الحالي على **محرك تخزين محلي متطور فائق الأداء (Offline-First)** مدمج بالكامل في متصفح الزائر:
- **خصوصية مطلقة**: كل منشور، رد، تفاعل، أو تعديل في الإعدادات يُحفظ مباشرة داخل متصفح المستخدم ولا يُرسل إلى أي خادم خارجي.
- **تهيئة تلقائية**: عند تشغيل التطبيق لأول مرة، يتم فحص التخزين وبدء حقن البيانات النموذجية من `src/data/seeds.js` لضمان واجهة ممتلئة وجذابة منذ اللحظة الأولى.
- **سهولة التطوير المستقبلي**: تم عزل منطق التخزين والقراءة بشكل يسهل استبدال طبقة التخزين المحلي بقاعدة بيانات سحابية حقيقية (مثل Cloud Firestore أو PostgreSQL) دون الحاجة لإعادة كتابة الواجهات الرسومية.

## 🚀 التشغيل والتطوير المحلي

لتشغيل المشروع على جهازك الشخصي، اتبع الخطوات التالية:

1. **تثبيت الحزم والاعتماديات**:
   ```bash
   npm install
   ```

2. **تشغيل سيرفر التطوير المحلي**:
   ```bash
   npm run dev
   ```
   *سيتم تشغيل الخادم المحلي على الرابط الافتراضي: `http://localhost:5173` أو `http://localhost:3000` حسب إعدادات المنفذ المتاحة.*

3. **بناء المشروع للإنتاج (Production Build)**:
   ```bash
   npm run build
   ```
